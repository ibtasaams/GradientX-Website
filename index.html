How it all fits together

The library is structured in clean, composable layers. Your model sits at the top and dispatches operations down to whichever compute backend is enabled.

Vulkan Backend
Platform-Agnostic
Platform-agnostic GPU compute using GLSL shaders dispatched via the Vulkan API. Works on any GPU — AMD, Intel, NVIDIA, and Apple Silicon. Best for cross-platform deployment. Enabled in the default build via use_GPU.h.
CUDA Backend
NVIDIA Only
High-performance compute kernels for NVIDIA GPUs. Targets Maxwell (52), Pascal (60/61), Volta (70), Turing (75), Ampere (80/86), and Hopper (90) architectures. Enable by uncommenting CUDA sources in CMakeLists.txt.
CPU Backends
No GPU Required
Two CPU parallelization options: a custom pthread-based multithreading implementation with hand-tuned workload distribution, and OpenMP integration for pragma-based parallelism. Both are SIMD-friendly.

Tensor
tensor.h
Core N-dimensional array class. Supports CPU and GPU memory with built-in display for debugging tensor contents and shapes.
Method Returns Description
matmul(a, b) Tensor Matrix multiply two tensors. Dispatches to GPU kernel when GPU mode is active.
softmax(x) Tensor Numerically stable softmax along last dimension.
argmax(x) Tensor Index of maximum value along specified axis.
cross_entropy(pred, target) float Cross-entropy loss between prediction logits and one-hot targets.
transpose(x) Tensor 2D matrix transpose. GPU version available.
display() void Pretty-prints tensor contents and shape to stdout.
Model
model.h
Sequential model container. Layers are added in order and executed sequentially. Handles the full training loop including forward pass, loss computation, backpropagation, and weight updates.
sequential_example.cpp
// Construct the model — pass true to enable GPU Model model(true); // Stack layers model.add(new Linear_GPU(128)); model.add(new ReLU()); model.add(new Linear_GPU(10)); // Train model.fit(labels, images, /*epochs=*/100, /*lr=*/0.01f); // Inference Tensor output = model.predict(test_input);
Method Signature Description
Model(gpu) bool Constructor. Pass true to enable GPU-backed layers.
add(layer) Layer* Append a layer to the stack. Accepts any layer that inherits from the base Layer class.
fit(labels, data, epochs, lr) void Run the full training loop. Computes forward pass, cross-entropy loss, backward pass, and gradient updates for each epoch.
predict(input) Tensor Forward pass only, no gradient computation. Returns raw logits.
Layers
layers.h
All layers inherit from a common base and implement forward_pass() and backward_pass(). GPU variants are suffixed with _GPU.
Layer GPU Variant Description
Linear(out) Linear_GPU Fully connected layer: y = xW + b. Weight and bias are learned via backprop.
Conv2D(filters, kernel) Conv2D_GPU 2D convolution with configurable filters and kernel size. Backward pass computes dW and dX.
MaxPool2D(size) — Spatial max pooling. Reduces spatial dimensions by the given pool size.
ReLU() — Element-wise rectified linear unit. Fast, simple, and works on CPU and GPU tensors.
Embedding(vocab, dim) — Learnable token embedding lookup table. Used in the transformer encoder/decoder.
MHA(heads, dim) — Multi-Head Attention. Computes scaled dot-product attention across multiple heads in parallel.
MultiHeadAttention
layers.h
Scaled dot-product attention mechanism split across multiple heads. Projects queries, keys, and values, computes attention scores, and concatenates head outputs. Core building block of the transformer.
attention_math.txt
// Attention formula (per head) Attention(Q, K, V) = softmax( Q·Kᵀ / √dₖ ) · V // Multi-head: run h heads in parallel then concat MultiHead(Q,K,V) = Concat(head₁,...,headₕ) · Wₒ where headᵢ = Attention(Q·Wᵢᵠ, K·Wᵢᴷ, V·Wᵢᵛ)
Functional Model
functional_model_template.cpp
For architectures that cannot be expressed as a simple stack of layers — such as encoder-decoder transformers, residual networks, or models with shared weights — use the functional model pattern.
Feature Description
Custom forward graph Define the exact execution order of layers, residual connections, and branching paths.
Shared weights The same layer instance can be referenced at multiple points in the graph.
Custom training loop Full control over loss computation, gradient accumulation, and optimizer step timing.
Example included transformer.cpp demonstrates a complete encoder-decoder architecture using this pattern.
GPU Pipeline
use_GPU.h
Wraps the Vulkan compute pipeline — instance creation, device selection, command buffer recording, and shader dispatch — behind a clean C++ interface. Shaders are written in GLSL and compiled to SPIR-V at build time.
Component File Description
VulkanPipeline use_GPU.h / .cpp Manages Vulkan instance, physical device, logical device, and compute queue.
GPU Layers layers_GPU.cpp Layer implementations that dispatch work to the Vulkan pipeline instead of the CPU.
GPU Matrix Ops matrix_ops_GPU.cpp Core math operations (matmul, etc.) implemented as Vulkan compute dispatches.
GLSL Kernels kernel.cu / shaders/ Compute shaders for matmul, convolution forward/backward, and element-wise ops.

Get running in minutes

Clone, configure your GPU backend, and build with CMake. No package manager, no Python environment.

Build Steps

1
Clone the repository
git clone https://github.com/gideon116/Cpp_ML.git
2
Navigate and create a build directory
cd Cpp_ML && mkdir build && cd build
3
(Optional) Edit CMakeLists.txt to set your Vulkan SDK path or enable CUDA. The SDK path variable is near the top of the file.
4
Configure, build, and run
cmake -DCMAKE_BUILD_TYPE=release .. && make && ./wef
5
To run a different example, edit train.cpp and call the desired function (e.g., transformer() or your own).
